SEED Platform v10.x - Quick Start

Get started with the SEED Platform in 5 minutes.

Prerequisites

  • Node.js 18+ installed
  • Docker and Docker Compose (optional, for deployment)

Quick Setup

1. Install Dependencies

cd seed-platform-v10x/scripts
npm install

2. Test with Example Bundles

The platform includes test bundles. Generate artifacts:

npm run send-share

Output:

send-share complete:
  sha256.txt: double-space format
  signatures.json: 64-hex siblings, 128-hex signatures

3. Verify Generated Artifacts

npm run verify

Expected Result:

‚úÖ ALL CHECKS PASSED (STRICT FORMAT)

What Just Happened?

  1. send-share.cjs scanned artifacts/bundles/ for .tar.gz files
  2. Generated artifacts/sha256.txt with exact double-space format
  3. Generated artifacts/signatures.json with Merkle proofs
  4. verify-bundle.ts validated all checksums, proofs, and signatures

Next Steps

Add Your Own Bundles

rm artifacts/bundles/*.tar.gz

Deploy with Docker

docker build -t seed10x-production:latest .

docker-compose up -d

# Check logs
docker-compose logs -f

Initial Setup

Step 1: Install Node.js Dependencies

cd seed-platform-v10x/scripts
npm install

This installs: - @noble/ed25519 - Ed25519 signature verification - qrcode - QR code generation - tsx - TypeScript execution

Step 2: Prepare Public Keys

Add your Ed25519 public keys to backend/keys/:

# Example: Create a public key file
echo "your-64-hex-public-key-here" > backend/keys/signer-master-2025.pub

Format Requirements: - Filename: signer-*.pub (e.g., signer-master-2025.pub) - Content: 64 hexadecimal characters (32 bytes) - No spaces or line breaks

Step 3: Prepare Bundle Files

Place your artifact bundles in artifacts/bundles/:

# Example: Copy your bundles
cp /path/to/your/bundle-v1.0.0.tar.gz artifacts/bundles/
cp /path/to/your/bundle-v1.0.1.tar.gz artifacts/bundles/

Supported Formats: - .tar.gz files only - Any size supported - Multiple bundles allowed

Generating Artifacts

Step 1: Run send-share Script

cd scripts
npm run send-share

Or directly:

node send-share.cjs

Step 2: Verify Generated Files

Check that the following files were created:

ls -lh ../artifacts/sha256.txt
ls -lh ../artifacts/signatures.json

Step 3: Inspect Output Format

sha256.txt format:

<64-hex-hash><two-spaces><filename>

Example:

cat ../artifacts/sha256.txt

Output:

724362907ca0ef2c8c82a2db6b859c4e2853318e2fd3f647a0c4c532bc8711d8  bundle-v1.0.0.tar.gz
a11150e0a63526853a0da340f47a750c58287ad8094f54d0a41c8d6305f79ef2  bundle-v1.0.1.tar.gz

signatures.json format:

cat ../artifacts/signatures.json | jq .

Key fields: - merkle_root: 64-hex root hash - bundles[].sha256: 64-hex file hash - bundles[].merkle_proof: Array of L/R siblings (64-hex each) - bundles[].signatures[].signatureHex: 128-hex signature

Verifying Bundles

Online Verification (Full System)

cd scripts
npm run verify

This performs three validation steps:

1. SHA256 Validation - Computes SHA256 of each bundle file - Compares against sha256.txt - Compares against signatures.json

2. Merkle Proof Validation - Verifies each bundle's proof path - Validates against merkle_root - Ensures proof structure integrity

3. Ed25519 Signature Validation - Loads public keys from backend/keys/*.pub - Verifies signatures against merkle_root - Validates k-of-n threshold

Docker Deployment

Step 1: Build Docker Image (if needed)

# If you have a Dockerfile
docker build -t seed10x-production:latest .

Or pull from registry:

docker pull your-registry/seed10x-production:latest

Step 2: Configure Environment

Edit docker-compose.yml if needed:

environment:
  - SEED_ARTIFACT_MODE=local
  - SEED_ARTIFACT_PATH=/app/artifacts
  - SEED_TIER1_AUTO_INITIATE=true
  - SEED_TIER2_AUTO_INITIATE=true

Step 3: Start Platform

docker-compose up -d

Step 4: Verify Running Container

docker-compose ps
docker-compose logs -f

Step 5: Access Platform

The platform runs on port 8000:

curl http://localhost:8000

Step 6: Stop Platform

docker-compose down

Troubleshooting

Issue: "No public keys found"

Error:

‚ùå No public keys found in backend/keys/*.pub

Solution: 1. Check that public key files exist: ls backend/keys/*.pub 2. Verify filename format: signer-*.pub 3. Verify content format: 64 hexadecimal characters 4. Example: echo "0123456789abcdef..." > backend/keys/signer-test.pub

Issue: "SHA256 mismatch"

Error:

‚ùå sha mismatch: bundle-v1.0.0.tar.gz

Solution: 1. Verify file integrity: sha256sum artifacts/bundles/bundle-v1.0.0.tar.gz 2. Regenerate artifacts: npm run send-share 3. Check for file corruption during transfer 4. Ensure no modifications after generation

Issue: "Merkle proof failed"

Error:

‚ùå merkle proof failed: bundle-v1.0.0.tar.gz

Solution: 1. Regenerate signatures.json: npm run send-share 2. Verify all bundles are present in artifacts/bundles/ 3. Check that no bundles were added/removed after generation 4. Ensure deterministic ordering (alphabetical)

Issue: "Invalid signatureHex"

Error:

‚ùå signatureHex must be 128 hex: seed-master-2025

Solution: 1. Verify signature format in signatures.json 2. Ensure 128 hexadecimal characters (64 bytes) 3. Check for proper Ed25519 signature generation 4. Regenerate with correct signing process

Issue: "Missing signatures.json"

Error:

‚ùå Missing /path/to/artifacts/bundles/signatures.json

Solution: 1. Run send-share script: npm run send-share 2. Check that bundles exist in artifacts/bundles/ 3. Verify write permissions on artifacts/ directory 4. Check for script execution errors

Issue: "Signature validation failed"

Error:

‚ùå seed-master-2025
Signatures valid: 0/1 required

Solution: 1. Verify public key matches private key used for signing 2. Check public key format (64 hex characters) 3. Verify signature was generated for correct merkle_root 4. Ensure Ed25519 algorithm compatibility

Issue: "Module not found"

Error:

Error: Cannot find module '@noble/ed25519'

Solution:

cd scripts
npm install

Issue: "Docker container won't start"

Error:

Error response from daemon: pull access denied for seed10x-production

Solution: 1. Build image locally: docker build -t seed10x-production:latest . 2. Or pull from correct registry: docker pull your-registry/seed10x-production:latest 3. Check image name in docker-compose.yml 4. Verify Docker registry authentication

Advanced Usage

Custom Artifact Root

Set ARTIFACT_ROOT environment variable:

export ARTIFACT_ROOT=/custom/path
npm run verify

Multiple Signers (k-of-n)

  1. Add multiple public keys to backend/keys/:
echo "pubkey1..." > backend/keys/signer-alice.pub
echo "pubkey2..." > backend/keys/signer-bob.pub
echo "pubkey3..." > backend/keys/signer-charlie.pub
  1. Set k_threshold in signatures.json:
{
  "merkle_root": "...",
  "k_threshold": 2,
  "bundles": [...]
}
  1. Verification requires at least k valid signatures

Best Practices

  1. Always verify after generation: Run npm run verify immediately after npm run send-share
  2. Keep keys secure: Never commit private keys to version control
  3. Use k-of-n signatures: Require multiple signers for production
  4. Automate verification: Integrate into CI/CD pipeline
  5. Document key management: Maintain clear records of key ownership
  6. Test air-gapped flow: Regularly test offline verification process
  7. Monitor bundle integrity: Set up alerts for verification failures
  8. Version control artifacts: Tag releases with corresponding bundle versions

Appendix: File Format Specifications

sha256.txt Format

<64-hex-hash><TWO-SPACES><filename>\n
  • Hash: Exactly 64 hexadecimal characters (lowercase)
  • Separator: Exactly TWO space characters (ASCII 32)
  • Filename: Original bundle filename
  • Line ending: Unix LF (\n)
  • No trailing whitespace

signatures.json Format

{
  "merkle_root": "64-hex-string",
  "k_threshold": 1,
  "n_signers": 3,
  "bundles": [
    {
      "file": "bundle-name.tar.gz",
      "sha256": "64-hex-string",
      "merkle_proof": [
        { "side": "L", "sibling": "64-hex-string" },
        { "side": "R", "sibling": "64-hex-string" }
      ],
      "signatures": [
        {
          "signerId": "signer-name",
          "signatureHex": "128-hex-string"
        }
      ]
    }
  ]
}
  • All hex strings: lowercase
  • merkle_root: 64 hex (32 bytes)
  • sha256: 64 hex (32 bytes)
  • sibling: 64 hex (32 bytes)
  • signatureHex: 128 hex (64 bytes)
  • side: "L" or "R" only

Public Key Format

<64-hex-string>
  • Filename: signer-*.pub
  • Content: 64 hexadecimal characters (lowercase)
  • Represents: Ed25519 public key (32 bytes)
  • No spaces, no line breaks
  • Example: 0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef
